fix: extend unauthenticated client fix to all remaining executors

Seven more executors were still injecting opts.Token into probes that
claim to represent unauthenticated access: task_idor (steps 2 & 3),
resources_unauth, prompt_unauth, oauth_dcr (registration POSTs),
registry_poison, session_smuggle (GetTask cross-context probe), and
delegation_escalation. All now use NewUnauthHTTPClient for the specific
requests where the finding narrative claims "no credentials".

Additional fixes from architecture and test reviews:

Engine hardening:
- engine.runOne cancels immediately if ctx is already done
- executor panics are caught via defer/recover and surfaced as RunResult.Err
  so the rest of the scan can continue

Auth / security:
- discoverTokenURLWithClient now rejects http:// token_endpoint values to
  prevent SSRF and cleartext credential transmission (separate from the
  existing issuer scheme check)
- FetchClientCredentialsToken and ExchangeAuthCode error messages no longer
  include response bodies, which could leak provider-specific debug info

Dead code / cleanup:
- Removed unused discoverMCPEndpoint from session.go (duplicate of
  initializeMCP logic in resources_unauth.go)
- Removed unused oobEnabled variable in cli/scan.go; deprecated the --oob
  flag since OOB behavior is fully controlled by --oob-url alone

CI quality gates:
- Pinned golangci-lint to v2.1.6 (was "latest")
- Pinned govulncheck to v1.1.4 (was "@latest")
- Release job now requires lint and vuln jobs to pass before publishing

Test coverage:
- attack/http_test.go: two new tests for NewUnauthHTTPClient/NewHTTPClient
  token behavior (uses new export_test.go TokenOf accessor)
- auth/oauth_test.go: updated to use fake HTTPS token_endpoint; added
  TestDiscoverTokenURL_RejectsHTTPTokenEndpoint
- report/printer_test.go: two new tests for RiskIndicator "[indicator]"
  tag and ConfirmedExploit (no tag) in PrintScanSummary output
- session_smuggle_test, task_idor_test, context_orphan_test,
  resources_unauth_test, extcard_test: Confidence assertions added
- Python SDK: run() test asserts confidence/is_confirmed; probe() now
  passes --config and has a corresponding test
